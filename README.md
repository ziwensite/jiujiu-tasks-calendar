# 久久任务日历 (JiuJiu Tasks Calendar)

一个功能全面的 Obsidian 日历插件，支持农历、节假日、任务管理和笔记管理功能，帮助您高效组织和管理日常工作与生活。

## 功能特点

### 🌙 农历与节假日
- 支持显示农历日期和节气
- 显示法定节假日和调休信息
- 支持节假日标记（休/班）

### 📅 日历视图
- 支持月视图和年视图切换
- 显示周数和星期标题
- 支持导航到上一月/下一月/上一年/下一年
- 快速返回今天的功能

### ✅ 任务管理
- 支持从笔记中提取任务
- 显示任务的截止日期和开始日期
- 支持任务完成状态切换
- 支持任务选中和双击操作
- 支持闪念和记录功能
- 支持与 tasks 插件格式兼容

### 🎯 自定义标签
- 支持 LB1 和 LB2 两个自定义标签
- 可配置标签文本和操作类型
- 支持系统命令和打开文件两种操作

### 📝 笔记管理
- 支持日记自动识别
- 支持周报和任务状态指示
- 支持与 Obsidian 笔记系统集成

## 安装方法（暂时未上架）

1. 打开 Obsidian 设置
2. 点击 "社区插件"
3. 关闭 "安全模式"
4. 点击 "浏览"
5. 搜索 "JiuJiu Calendar"
6. 点击 "安装"
7. 安装完成后点击 "启用"

### 手动安装

1. 下载最新的发布版本（.zip 文件）
2. 解压文件到 Obsidian Vault 的 `.obsidian/plugins/` 目录下
3. 确保解压后的文件夹名称为 `jiujiu-tasks-calendar`
4. 重启 Obsidian
5. 打开 Obsidian 设置 → 社区插件
6. 找到 "久久任务日历" 并启用

### 从源码构建

1. 克隆仓库到本地
2. 进入插件目录，执行 `npm install` 安装依赖
3. 执行 `npm run build` 构建插件
4. 将构建后的文件复制到 Obsidian Vault 的 `.obsidian/plugins/jiujiu-tasks-calendar/` 目录
5. 重启 Obsidian 并启用插件

## 使用说明

### 日历操作
- **点击日期**：选择日期，显示该日期的任务
- **双击日期**：创建或打开该日期的日记
- **点击周数**：选择整周，显示该周的任务
- **双击周数**：创建或打开该周的周报
- **点击月份**：切换到月视图
- **点击年份**：切换到年视图

### 任务操作
- **点击任务**：选中任务
- **双击任务**：打开任务所在的笔记
- **点击复选框**：编辑任务内容及状态

### 标签操作
- **点击 LB1/LB2**：执行配置的操作（系统命令或打开文件）
- **设置标签**：在插件设置中配置标签的文本和操作类型

## 支持的任务格式

插件支持多种任务格式，包括 tasks 插件的格式：

### 基本任务格式
```
- [ ] 任务内容
- [x] 已完成任务
```

### 日期格式
- 完整支持 tasks 插件的日期格式
- 截止日期：`@YYYY-MM-DD`, `#YYYY-MM-DD`, `📅 YYYY-MM-DD`, `due: YYYY-MM-DD`
- 开始日期：`🛫 YYYY-MM-DD`, `start: YYYY-MM-DD`, `🔄 YYYY-MM-DD`

### 全天标记
```
- [ ] 全天任务 全天
- [ ] all day task all day
- [ ] full day task full day
- [ ] recurring task 🔄
```

### 时间范围
```
- [ ] 任务内容 10:00-12:00
- [ ] 任务内容 10:00 ~ 12:00
- [ ] 任务内容 10:00 - 12:00
```

### 单个时间点
```
- [ ] 任务内容 10:00
- [ ] 任务内容 10:00 AM
- [ ] 任务内容 10:00 PM
```

## 设置说明

插件提供了丰富的配置选项，您可以在 Obsidian 设置 → 久久任务日历 中进行调整：

### 🌙 农历与节假日
- **显示农历**：控制是否在日历中显示农历日期
- **显示节气**：控制是否显示二十四节气
- **显示节假日**：控制是否显示法定节假日和调休信息

### 📅 日历视图
- **默认视图**：设置首次打开日历的默认视图（月视图/年视图）

### ✅ 任务管理
- **任务格式支持**：选择支持的任务格式（tasks 插件格式兼容）
- **任务提取范围**：设置从哪些文件中提取任务
- **日记自动识别**：启用日记自动识别功能
- **周报状态指示**：启用周报和任务状态指示

### 🏷️ 自定义标签
- **LB1/LB2 启用**：控制是否显示自定义标签
- **标签文本**：设置标签显示的文字
- **操作类型**：
  - **系统命令**：执行 Obsidian 系统命令
  - **打开文件**：打开指定的笔记文件
- **命令/文件路径**：根据操作类型设置对应的命令或文件路径

### 📝 笔记管理
- **文件夹**：设置日记、周报、季报、年报存储的文件夹
- **周报格式**：配置日记、周报、季报、年报的命名格式

### 🎨 外观设置
- **主题适配**：自动适配 Obsidian 主题
- **自定义样式**：支持通过 CSS 自定义样式

### ⚡ 性能优化
- **缓存设置**：调整数据缓存策略
- **更新频率**：设置日历数据的更新频率

## 开发和构建

### 开发环境
- Node.js 16.0.0 或更高版本
- npm 或 yarn 包管理器

### 开发流程

1. **克隆仓库**
   ```bash
   git clone https://github.com/yourusername/jiujiu-tasks-calendar.git
   cd jiujiu-tasks-calendar
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **开发模式**
   ```bash
   npm run dev
   ```
   - 自动监听文件变化，实时构建
   - 构建产物输出到 `dist/` 目录

4. **生产构建**
   ```bash
   npm run build
   ```
   - 生成优化后的生产版本
   - 构建产物输出到 `dist/` 目录

5. **代码检查**
   ```bash
   npm run lint
   ```
   - 检查代码是否符合 ESLint 规范

### 项目结构
```
├── src/             # 源代码目录
│   ├── components/  # UI 组件
│   ├── core/        # 核心功能
│   ├── engine/      # 引擎逻辑
│   ├── modals/      # 弹窗组件
│   ├── services/    # 服务层
│   ├── types/       # TypeScript 类型定义
│   ├── utils/       # 工具函数
│   └── views/       # 视图组件
├── styles.css       # 样式文件
├── main.ts          # 插件入口
└── manifest.json    # 插件配置
```

## 常见问题 (FAQ)

### Q: 插件支持哪些任务格式？
A: 插件支持多种任务格式，包括 Obsidian 原生任务格式和 tasks 插件格式，具体支持的格式请参考 [支持的任务格式](#支持的任务格式) 部分。

### Q: 如何自定义日历样式？
A: 您可以通过 Obsidian 的 CSS 代码片段功能来自定义日历样式，插件提供了丰富的 CSS 类名供您使用。

### Q: 插件会影响 Obsidian 的性能吗？
A: 插件经过优化，默认配置下不会对 Obsidian 性能产生明显影响。您可以通过调整任务提取范围和更新频率来进一步优化性能。

### Q: 如何备份插件设置？
A: 插件设置会自动保存到 Obsidian 的配置文件中，您可以通过备份 Obsidian 的 `.obsidian` 文件夹来备份插件设置。
